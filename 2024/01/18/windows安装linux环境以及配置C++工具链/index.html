Windows 安装 Linux 以及配置 C++工具链

港大 ENGG1340 课程内里有包含 Linux 和 C++相关内容. 介于有很多同学都是小萌新和小白, 不知道如何在 windows 环境下配置 linux 以及 c++. 于此写一篇攻略

首先我们了解一个概念. windows 和 linux 分别是两套操作系统, 我们在 1340 上学习的应该是 linux 世界里的 Shell, 一门编程语言. 这和 window 上的 powershell 以及 cmd (Command Line) 是不一样的. 就比如命令上的差异:

powershell shell cmd 作用
Get-ChildItem ls dir 列出当前目录
New-Item -ItemType Directory -Path C:\NewDirectory mkdir NewDirectory mkdir NewDirectory 创建新目录
Copy-Item -Path C:\Source\File.txt -Destination C:\Destination\ cp Source/File.txt Destination/ copy C:\Source\File.txt C:\Destination\ 复制文件
Remove-Item -Path C:\Path\To\File.txt rm Path/To/File.txt del C:\Path\To\File.txt 删除文件
Move-Item -Path C:\Source\File.txt -Destination C:\Destination\ mv Source/File.txt Destination/ move C:\Source\File.txt C:\Destination\ 移动文件或目录

所以你没法在 windows 上的 powershell 或是 cmd 中运行 shell 的命令. 这也是为什么你需要一个 linux 系统.

以下是两篇我觉得比较好的链接, 帮助你了解各种关于命令行, 终端, 内核 等概念.

reference:
  • What is Terminal, Console, Shell and Kernel? - GeeksforGeeks
  • The Shell and the Terminal: What’s the Difference? | HackerNoon

1.2 使用 VMWare 安装

VMWare 是一款商业化的高性能, 高度配置的虚拟机. 一般企业用的比较多. 付费嘛~, 算什么大问题, 咱百度上啥资源没有你说是吧(doge). 破解版随便搜一下就可以了.

介于 VMware 与 HyperV 会有冲突问题, 我把解决方案的文章链接也贴下面了

安装文档:
  • 2020 最新版 VMware 安装 Ubuntu20.04 教程(巨细)！ - 知乎 (zhihu.com)

注: 本人使用的是 VMWare 17 Pro, 好像 VM 在 15.5 后就把冲突兼容问题给解决了…

在安装好虚拟机后你就可以拥有半台自己的 linux 操作系统的电脑了, 为什么算半台? 因为这玩意占你本机资源, 撑死算个双胞胎兄弟.

在你安装好 ubuntu, 配置到系统后. 在系统里面按 Ctrl + Alt + t 就可以打开一个终端, 在里面你就可以尽情的尝试 shell 脚本啦

2. 安装 WSL (Windows Subsystem Linux)

wsl 就是一个阉割版的 linux 子系统, 在性能上会有所欠缺, 但好处就是轻量, 快速. 对于仅仅想学习 linux 命令的同学, 这个会是个非常不错的选择. 但是对于以后想搞运维, 需要深入理解 linux, 亦或者学习跨平台开发, 远程服务器之类的, 搞个虚拟机, 不亏. (注: 不嫌麻烦的话可以学习本人, 两个都有, 既有虚拟机也有 wsl)

安装文档:
  • 最详尽教程完整介绍-Windows 的 Linux 子系统-WSL1&WSL2-腾讯云开发者社区-腾讯云 (tencent.com)

安装成功后在终端里敲 bash 就可以进入 linux 子系统的终端啦 (记得要运行该系统, 不然系统死的你怎么搞都进不去)

注: 由于 wsl2 默认安装在 C 盘, 很多人不希望杂七杂八的玩意儿都塞 C 盘, 这里就附上一则教你怎么迁移 wsl 的教程

迁移 wsl:
  • 【WSL】WSL 迁移教程-CSDN 博客

2. Windows 配置 C++

windows 上配置 c++ 相关的工具链其实也挺简单的, 虽然我当时糊弄了好久… 想要编写 C++和 C 语言, 你必须在你的电脑上安装这俩程序的编译器, 也就是 gcc , (g++ 也可以奥, g++其实就是 gcc 的升级版, 专门为 C++ 做了适配的), 以及 gdb 用来 debug 修代码的.

由于 windows 上没有原生的包管理工具, 所以你的自己手动下载一下, 可能还得自己配一下环境变量. 这时候有人问, 为啥不直接在虚拟机上耍啊, linux 自带 gcc 编译器. 没问题啊, 你完全可以在虚拟机上写代码. 但对本人来说, 电脑配置比较吃紧(虚拟机分配的不多), 而且虚拟机上没有你辛辛苦苦用了好久的 IDE, 比如我的 VS Code. 里面好多好用的插件呢. 在虚拟机上搞得话得重新下载. 而且虚拟机还吃资源, 卡. 所以本人倾向于在 windows 整个 gcc, 然后在本地敲代码. 美滋滋~

那接下来话不多说, 扒拉扒拉如何安装 C/C++ 工具链

1. 配置 C++编译器

1.1 使用 MinGW-w64

MinGW 说人话就是一个能在 Windows 下运行的二进制文件, 不依赖任何非 windows 的东西. 但是 mingw 本身不带有 linux shell 环境, 一些常用的编译前处理命令工具没法使用. 而 MSYS2 就是提供了一个 shell 环境, 然后支持安装 mingw. (注: 本人用的就是 MYSY2). 相较于 MinGW 要大一点, 多了个 shell 脚本环境.

安装文档:
  • MinGW-W64 下载、安装与配置（支持最新版的 GCC，目前 GCC 13.2.0）-CSDN 博客

1.2 使用 MSYS2

安装文档:
  • 给萌新的 C/C++环境搭建攻略（VSCode 和 MSYS2） - 知乎 (zhihu.com)
配置环境变量:

右击此电脑打开属性

单击高级系统设置 -> 选择环境变量 -> 找到系统变量里的 PATH 内并进入

单击新建, 将你们下载的 wingw64 的安装路径填进去, 记得是你们自己的安装路径!!! , 安装路径记得带有 bin 目录 (bin 表示 binary, 存放的是所有二进制文件, 实际的 g++, gcc, gdb 程序都在那里)

然后疯狂点确定就好了

安装好后配置环境变量后就可以在 windows 终端内使用 c++编译器啦
