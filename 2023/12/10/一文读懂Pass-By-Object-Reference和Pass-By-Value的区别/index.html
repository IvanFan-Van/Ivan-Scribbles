那么问题来了, 既然传入不可变变量又或是可变变量, python 和 Java 的处理都相同. 为什么一个是 pass-by-referencce 一个是 pass-by-value ?
这两个到底有什么区别. 现在就来揭晓.

之所以有这样的不同, 是因为 python 和 Java 语言本身导致的不同. 在 python 中是没有基础数据类型的. 所有的一切都是 object 对象. 也就是说, 不管是不可变的 Number 还是 String 也好. 还是可变的 list 或是 map 也罢. 在 python 中都是一个对象, 而存储在变量内的其实是它们在内存中的 16 进制的地址, 也就是类似这样的东西: 0x7ff6b2c03cf8. 你使用 python 保存一个数字, 最后存储在变量的内容应该类似这样: a = 10, a -> 0x7ff6b2c03cf8

而在 Java 中, 数据分为基础数据类型 Primitive Types 和 引用类型 Reference Types. 也就是说, 在 Java 中使用变量保存的数字, 在变量中就是这个数字本身, 比如: int num = 10. 那么 num 就是 10, 而不是一串十六进制地址. 所以说, 在 Java 中传递参数, 如果参数类型为基础数据类型, 则传递的是基础数据类型的拷贝, 这也是为什么在第一个例子中对参数重新赋值并不会改变原有的值. 而传递的参数类型如果是引用类型, 则传递的是引用的拷贝, 也就是一个指向同一块内存的指针.

而在 python 中, 所有的参数都是引用的拷贝, 指向参数指向的那一块内存.

总结: python 和 Java 在参数传递上最终的处理结果是类似的, 但实际参数保存的内容是不同的. 在 python 中所有数据都是对象, 所以所有变量都是一个十六进制的内存地址. 而在 Java 中, 当数据类型为基础数据类型时参数是其本身的拷贝. 而参数为引用数据类型时, 传入的参数才是十六进制的内存地址
